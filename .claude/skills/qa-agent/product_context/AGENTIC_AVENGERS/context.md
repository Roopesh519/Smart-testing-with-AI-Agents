- **OTP flow**: Login → email OTP (real code sent to registered email). The OTP fields are React-controlled inputs (`maxlength="1"` per box); programmatic injection via JS/clipboard doesn't work — must be entered manually or via `human_type` after clicking field 1 and letting focus auto-advance.
- **React textarea**: The chat input is a controlled React `<textarea>` with no HTML `maxlength` attribute. Character limit (5000) is enforced at React state level and displayed as "N / 5000" counter below the input. Programmatic paste/value injection does not work — use `human_type` for realistic input.
- **Session naming**: System auto-generates session title after first query. Sessions labelled "Unable to generate title - insufficient context" appear in sidebar when naming AI fails.
- **Out-of-scope handling**: Queries outside workspace scope return a redirect message rather than an error — graceful UX.
